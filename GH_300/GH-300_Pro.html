📘 Part 1 -- Fundamentals

Six Principles of Responsible AI

  • Fairness -- Treat all people fairly; detect bias via diverse data, adversarial debiasing, monitoring across segments
  • Reliability & Safety -- Perform consistently, respond safely to unexpected conditions, resist manipulation
  • Privacy & Security -- Get user consent, collect only needed data, anonymize (pseudonymization/aggregation), encrypt with HSMs
  • Inclusiveness -- Accessible to all abilities, languages, geographies; support screen readers, voice control, offline use
  • Transparency -- Explain how systems operate, justify design choices, be honest about limitations, enable auditing
  • Accountability -- Humans responsible for AI operations; continuous monitoring, risk mitigation

Managing AI Risks

  • • Always review AI-generated code before deploying
  • • Minimize bias with diverse training data and fairness metrics
  • • Validate security to prevent unauthorized access and data breaches
  • • Provide clear explanations of what AI is doing (transparency)
  • • Maintain human oversight for compliance requirements
  • • GitHub Copilot may suggest deprecated features from training data
  • • Pattern-based responses -- doesn't verify computation accuracy

What is GitHub Copilot?

  • • AI pair programmer powered by OpenAI Codex/GPT models
  • • 46% of new code written by AI, 55% faster productivity
  • • Available in VS Code, Visual Studio, JetBrains, Neovim
  • • Provides inline suggestions + chat + CLI assistance
  • • Trained on public source code with broad language knowledge

Key Features

  • • Copilot Chat -- Ask questions, get explanations, generate tests
  • • PR Summaries -- Auto-generate pull request descriptions
  • • Code Review -- Suggest issues, describe changes, propose improvements
  • • CLI -- Command suggestions, shell scripts, error understanding
  • • Copilot Spaces -- Project-wide context for planning & design
  • • Cloud Agent -- Autonomous multi-step coding tasks

Subscription Plans

  • • Free -- Limited completions & chat, AI model access
  • • Pro -- Higher limits, priority models, advanced suggestions
  • • Pro+ -- Premium capacity, priority infrastructure
  • • Business -- Centralized management, policies, IP indemnity, public code filter
  • • Enterprise -- All Business + personalized suggestions, knowledge bases, custom models, codebase indexing

Interacting with Copilot

  • • Inline suggestions: Tab to accept, Esc to reject
  • • Chat panel: Natural language questions about code
  • • Inline chat: Ctrl+I for context-specific help
  • • Comments->Code: Describe in comments, get implementation
  • • Slash commands: /explain, /tests, /suggest, /comment
  • • Multiple suggestions: Alt+] to cycle alternatives

Setup & Configuration

  • • Install extension from VS Code Marketplace
  • • Sign in with GitHub account that has Copilot access
  • • Enable/disable via status bar icon
  • • Configure per-language in Settings -> Extensions -> Copilot
  • • Troubleshoot: Developer Tools, Command Palette -> Diagnostics

Data & Privacy

  • • Business/Enterprise: data NOT used for model training
  • • Individual: opt-out available for telemetry
  • • Proxy service hosted on Microsoft Azure
  • • Duplication detection (~150 char matching vs public code)
  • • Content exclusion configured in Copilot settings UI
  • • Status bar shows content exclusion status

Prompt Engineering Foundations

  • • Zero-shot: Give only a question/task, no examples
  • • Few-shot: Provide examples of expected input/output
  • • Role prompting: Assign a role for contextual responses
  • • Be specific -- clearly define the problem or task
  • • Use meaningful variable names for better context
  • • Provide examples of desired output in prompts
  • • Break complex tasks into smaller, focused prompts

Context & Process Flow

  • • @workspace -- Incorporate project-wide context
  • • #file / #editor -- Anchor to specific files/editors
  • • FIM (Fill-in-the-Middle) -- Uses prefix + suffix for better completions
  • • Pre-processing enriches prompt with additional context
  • • Post-processing: quality checks, content filtering, public code matching
  • • Chat history provides personalized suggestions
  • • Context from: open tabs, cursor location, neighboring files

LLMs & How Copilot Works

  • • Powered by Large Language Models (GPT-based)
  • • Trained on public code -- reflects common patterns
  • • Suggests code based on probability, not correctness
  • • May suggest deprecated syntax from training data
  • • Cannot verify mathematical computations
  • • Tends toward homogeneous solutions from popular patterns
  • • Quality depends on prompt clarity and context provided

Best Practices

  • • Review and refactor suggestions regularly
  • • Use /fix for syntax errors, /optimize for performance
  • • Keep relevant files open for better context
  • • Accept next word (partial accept) via shortcuts
  • • Check alternative suggestions in new tabs
  • • Validate security of all generated code
  • • Use Chat for explain, test generation, refactoring

IDE Features

  • • Inline code completions (single/multi-line)
  • • Chat interface for explanations & test generation
  • • Inline chat (Ctrl+I) for targeted modifications
  • • Available in all commercially supported IDEs (Business+)
  • • Keyboard shortcuts: Tab (accept), Alt+] (cycle), Esc (dismiss)

GitHub.com & Enterprise

  • • PR summaries (Enterprise, github.com only)
  • • Knowledge bases for org-specific content
  • • Content: code snippets, design patterns, documentation
  • • NOT: compiled binaries, screenshots
  • • Custom models use patterns from your repositories
  • • Codebase indexing for personalized suggestions

Management & Policies

  • • Policies: Organization/Enterprise settings -> Copilot -> Policies
  • • Content exclusion: Copilot settings UI (not .copilotignore)
  • • Duplication detection configurable at org & enterprise level
  • • Audit logs track admin activities and policy changes
  • • REST API: seat assignments, usage summaries, acceptance rates
  • • GitHub Copilot for Azure DevOps (no Enterprise license needed)

Developer Use Cases & SDLC

  • • Design phase: suggests patterns & best practices
  • • Coding: generation, completion, refactoring
  • • Testing: unit test generation, edge case discovery
  • • Review: PR summaries, code explanations
  • • Improves readability, reduces complexity
  • • Cannot autonomously refactor entire codebases

Test Generation

  • • Use /tests command or "Generate Unit Tests" from command palette
  • • Select function/class -> Copilot generates test cases
  • • Covers assertions, edge cases, boundary conditions
  • • Chat can explain test patterns and testing best practices
  • • Always review generated tests for correctness

Testing Best Practices

  • • Provide clear function signatures for better test generation
  • • Use descriptive test names for maintainability
  • • Include both happy path and error scenarios
  • • Copilot learns from your testing patterns over time
  • • Combine inline chat + test generation for comprehensive coverage
Spaces

Copilot Spaces -- Curated Context

What is a Copilot Space?
  • • A dedicated Copilot chat grounded in a curated set of context you choose
  • • Unlike general chat (broad), Spaces trade breadth for depth -> more predictable, grounded responses
  • • You control: files, issues, PRs, and free-text instructions
  • • Linked files reflect the repo's default branch (auto-updates)
Creating a Space:
  • • Go to github.com/copilot/spaces -> Create space
  • • Owned by you OR by an organization (org-owned can be shared via GitHub permissions)
  • • Add context: Attach files/folders, link PRs/issues, upload files, add free text
  • • Add Instructions: goals, style preferences, canonical examples
Best Practices:
  • • Keep Spaces small and focused (model context limits apply)
  • • Lead with most important sources (ordering affects responses)
  • • Use for: runbooks, playbooks, specific services, known datasets
  • • Instructions: brief, focused, actionable; include examples
IDE

Advanced Features -- Slash Commands & Chat

Ghost Text (Inline Suggestions):
  • • Suggestions appear as ghost text -> accept with Tab, ignore by typing
  • • Uses open files as context by default (no prompt required)
  • • Can provide prompts via comments, chat window, or inline chat
Chat Interfaces:
  • • Chat Pane: Click chat icon in sidebar -> dedicated panel for questions
  • • Inline Chat: Ctrl+I (Win) / Cmd+I (Mac) -- interact without leaving code context
  • • Inline chat benefit: suggestions happen closer to the code, no context switch
Slash Commands:
  • • /fix -- Fix bugs in selected code
  • • /doc -- Add comments/documentation
  • • /explain -- Get explanations about code
  • • /generate -- Generate code for a question
  • • /optimize -- Improve runtime performance
  • • /tests -- Create unit tests for selected code
  • • /help -- Get help on using Copilot Chat
Agents (Participants):
  • • @terminal -- Help with terminal errors and commands
  • • @workspace -- Questions about the whole project/workspace
  • • Agents provide domain-specific context for better responses
Language-Specific Tips (JS/Python):
  • • Copilot excels with Python and JavaScript due to large training corpus
  • • Use Copilot to: create Dockerfiles, set up virtual envs, write test suites
  • • Provide clear context: open related files, use descriptive variable names
  • • Combine slash commands with inline chat for best workflow
Admin

Management, Customization & Content Exclusions

Content Exclusions:
  • • Configure in Organization settings (NOT via .gitignore)
  • • Prevents Copilot from accessing/suggesting content from specified files/paths
  • • Available on Business and Enterprise plans only
  • • Applies to both completions and chat context
Public Code Matching:
  • • Detects and blocks suggestions that match public code verbatim
  • • Can be configured at org/enterprise level via policies
  • • Available on all plans (toggle in individual settings or org policy)
  • • GitHub provides contractual IP indemnification (Business/Enterprise)
Common Troubleshooting:
  • • No suggestions? Check: extension enabled, authenticated, not excluded file, network/proxy issues
  • • Poor suggestions? Provide more context, use meaningful names, add comments describing intent
  • • Chat not responding? Verify plan supports chat, check IDE compatibility
🏢 Plans & Administration
Plans

Copilot Plans & Pricing

Plans Available:
  • • Copilot Free: Limited access, 2000 completions/month, 50 chat messages/month
  • • Copilot Pro ($10/mo): Unlimited completions, premium models, cloud agent, 300 premium requests/month
  • • Copilot Pro+ ($39/mo): Everything in Pro + 1500 premium requests, all advanced models
  • • Copilot Business ($19/seat/mo): Centralized management, policy control, 300 premium requests/user
  • • Copilot Enterprise ($39/seat/mo): All Business features + enterprise-grade capabilities, 1000 premium requests/user
Key Differences (Business vs Enterprise):
  • • Organization-wide policy management: Business ✓ Enterprise ✓
  • • Content/file exclusion: Enterprise only
  • • Audit logs: Business ✓ Enterprise ✓
  • • Organization custom instructions: Enterprise only
  • • Copilot pull request summaries: Both
Enterprise

Enterprise Administration & Policies

Policy Management:
  • • Enterprise owners define policies for the whole enterprise OR delegate to org owners
  • • Policies control: feature availability, model access, agent permissions, MCP server access
  • • Navigate: Enterprise -> AI Controls -> Copilot/Agents/MCP sidebar
  • • Policies configured via dropdown menus or toggles
Access Management:
  • • Enterprise owners specify which organizations can use Copilot
  • • Organization owners specify which members can use Copilot
  • • Enterprise can assign Copilot Enterprise or Business to individual organizations
Monitoring & Compliance:
  • • Usage data: Review Copilot usage within org/enterprise to manage access and drive adoption
  • • Audit logs: Track actions taken and by which users
  • • File exclusions: Configure Copilot to ignore certain files (via org settings, not .gitignore)
Features

GitHub Copilot Features Overview

Assistive Features (synchronous):
  • • Copilot Chat: Available on GitHub.com, Mobile, IDEs (VS Code, Visual Studio, JetBrains, Eclipse, Xcode), Windows Terminal
  • • Inline Suggestions: Autocomplete-style code completions + Next Edit Suggestions (VS Code, Xcode, Eclipse)
  • • Pull Request Summaries: AI-generated PR descriptions and review focus areas
  • • GitHub Desktop: Auto-generate commit messages based on changes
Agentic Features (autonomous, need approval for sensitive actions):
  • • Copilot CLI: Use Copilot in terminal to add features, fix bugs, create PRs
  • • Cloud Agent: Autonomous AI that researches repos, creates plans, makes code changes on branches
  • • Code Review: AI-generated code review suggestions on PRs
  • • Agent Mode in IDEs: Copilot works autonomously -- determines files, offers changes, runs terminal commands
Customization Features:
  • • Custom Instructions: Repository-level or personal (all plans) vs Organization-level (Enterprise only)
  • • Prompt Files: Markdown files stored in workspace for reusable prompts
  • • MCP Servers: Connect external tools/data via Model Context Protocol
  • • Copilot Spaces: Organize code, docs, specs into Spaces for grounded context
  • • Block Public Code: Available on all plans -- filters suggestions matching public repos
📗 Part 2 -- Advanced & Agents
Agent

Agent Mode -- Autonomous Development

What is Agent Mode?
  • • An autonomous peer programmer -- goes beyond autocomplete to handle full tasks
  • • Analyzes entire codebase, determines relevant files and dependencies before making changes
  • • Works in iterative cycles: analyze -> suggest -> execute -> monitor -> refine
  • • Can create apps from scratch, refactor across files, write/run tests, migrate legacy code
Copilot Interaction Modes:
  • • Inline Suggestions: Real-time autocomplete as you type
  • • Copilot Chat: Dedicated chat panel with project-aware responses
  • • Copilot Edits: Apply changes across multiple files for structured modifications
  • • Agent Mode: Orchestrates dynamically, iterates to improve accuracy, handles complex workflows
Key Capabilities:
  • • Autonomous: Creates routes, updates configs, installs deps, generates tests -- without step-by-step instructions
  • • Multi-step orchestration: Draft -> Review -> Accept workflow
  • • Self-healing: Detects failures, applies fixes, re-runs until passing
  • • Context-aware: Recognizes project type via package.json, runs appropriate build scripts
  • • Developer retains full control: Review, adjust, or revert any proposed change
Limitations:
  • • May struggle with specialized domain logic or nuanced business rules
  • • Missing project context = less accurate solutions
  • • Premium reasoning doubles PRU consumption (~4+ vs ~2 standard)
Cloud

Copilot Cloud Agent

What is Cloud Agent?
  • • Autonomous AI agent that researches a repo, creates implementation plans, and makes code changes on a branch
  • • You can assign a GitHub issue to Copilot or ask it to open a PR directly
  • • Works on GitHub.com (not IDE-based like Agent Mode)
  • • Requires: Copilot Pro, Pro+, Business, or Enterprise; repos hosted on GitHub
Workflow:
  • • Assign an issue to Copilot -> it creates a branch and PR
  • • Track sessions via PR comments and activity
  • • Iterate with @copilot comments on the PR
  • • Review diff, approve, and merge when ready
Customization & Extension:
  • • Preconfigure agent's environment for your project
  • • Extend capabilities with MCP (Model Context Protocol)
  • • Test and validate output before merging
  • • Apply responsible-use practices and scope tasks effectively
MCP

MCP Server (Model Context Protocol)

What is MCP?
  • • Model Context Protocol -- a secure, scalable way to integrate GitHub features into AI tools
  • • Gives Copilot access to external tools and data sources
  • • Available on ALL Copilot plans
Setup & Configuration:
  • • Configure GitHub MCP Server in VS Code (or other MCP-compatible editors)
  • • Requires a GitHub Personal Access Token (PAT) for advanced setup
  • • Docker for local server setup (optional, for hands-on control)
  • • For orgs/enterprises: "MCP servers in Copilot" policy must be enabled
Use Cases:
  • • Use with Copilot Chat to automate development tasks
  • • Connect to databases, APIs, internal tools
  • • Extend Cloud Agent's capabilities
  • • Enterprise policy controls MCP access in Copilot (separate from third-party MCP hosts like Cursor/Windsurf)
Review

Code Reviews & Pull Requests with Copilot

Copilot Code Review Features:
  • • Request Copilot as a reviewer on GitHub.com PRs
  • • "Review selection" in VS Code (available on all plans including Free)
  • • Full code review (Business/Enterprise): catches issues, enforces best practices
  • • Supports custom instructions for project-specific review criteria
Capabilities:
  • • Accelerates review cycles by pre-screening code
  • • Catches common bugs, security issues, and style violations
  • • Enforces team best practices automatically
  • • Copilot PR summaries: AI-generated PR descriptions showing what changed and review focus areas
Automation:
  • • Automate reviews across repositories with rulesets and status checks
  • • Premium Request Units (PRUs) for deeper, context-rich analysis
  • • Always combine with human judgment and testing (Copilot reviews are advisory)
